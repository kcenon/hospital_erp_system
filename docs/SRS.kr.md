## 1. 소개 (Introduction)

### 1.1 목적 (Purpose)

본 문서는 **입원환자 관리 ERP 시스템**의 소프트웨어 요구사항을 상세히 기술합니다. 이 문서는 다음 대상에게 시스템의 기능적, 비기능적 요구사항을 전달하는 것을 목적으로 합니다:

- **개발팀**: 시스템 설계 및 구현의 기준
- **프로젝트 관리자**: 프로젝트 범위 및 일정 관리
- **품질 보증팀**: 테스트 케이스 도출 및 검증
- **이해관계자**: 요구사항 검토 및 승인

> **추적성 참조**: [PRD.md](PRD.md) 섹션 1.1-1.3

### 1.2 범위 (Scope)

#### 1.2.1 제품명
**입원환자 관리 ERP 시스템** (Inpatient Management ERP System)

#### 1.2.2 제품 목표

| ID | 목표 | 측정 지표 |
|----|------|----------|
| **OBJ-01** | 기존 진료 프로그램과 실시간 데이터 연동 | 데이터 동기화 지연 < 5분 |
| **OBJ-02** | 병실 현황 및 환자 상태 실시간 모니터링 | 현황판 갱신 < 3초 |
| **OBJ-03** | 회진/라운딩 보고의 모바일 지원 | 모바일 접속률 > 80% |
| **OBJ-04** | 의료법 및 개인정보보호법 준수 | 보안 감사 100% 통과 |

> **추적성 참조**: [PRD.md](PRD.md) 섹션 3, [프로젝트-개요.md](reference/01-overview/project-overview.md) 섹션 1.3

#### 1.2.3 제품 범위

```
입원환자 관리 ERP
├── 범위 내 (In Scope)
│   ├── 시스템 연동 (EMR/OCS, LIS)
│   ├── 병실 및 환자 관리
│   ├── 입퇴원 관리
│   ├── 보고서 및 일지
│   ├── 라운딩 관리
│   ├── 검사 결과 조회 (LIS 연동)
│   └── 관리자 기능
│
└── 범위 외 (Out of Scope)
    ├── 외래환자 관리
    ├── 수술실 관리
    ├── 재고/약품 관리
    ├── 청구/수납 시스템
    └── PACS 영상 직접 표시 (Phase 3+ 링크 연동 가능)
```

### 부록 D: 요구사항 ID 체계

```
요구사항 ID 형식: REQ-{카테고리}-{번호}

카테고리:
- FR    : Functional Requirement (기능 요구사항)
- NFR   : Non-Functional Requirement (비기능 요구사항)
- REG   : Regulatory Requirement (법규 요구사항)
- I18N  : Internationalization (국제화 요구사항)
- DOC   : Documentation (문서화 요구사항)

번호: 3자리 숫자 (001-999)

예시:
- REQ-FR-001  : 첫 번째 기능 요구사항
- REQ-NFR-010 : 열 번째 비기능 요구사항
- REQ-REG-001 : 첫 번째 법규 요구사항
```
